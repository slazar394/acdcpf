## Introduction

ACDCPF is a Python library for power flow analysis in hybrid AC/DC networks. It implements the **sequential AC/DC power flow method** based on the algorithms described in Beerten's work on VSC-HVDC systems.

### Key Features

- **Multi-terminal VSC-HVDC systems** with various converter control modes
- **DC-DC converters** for interconnecting DC grids of different voltage levels
- **Multiple independent DC grids** with proper voltage control coordination
- **Six VSC control modes**: P-Q, P-Vac, Vdc-Q, Vdc-Vac, Droop-Q, Droop-Vac
- **DC loads and generators** for modeling DC-side equipment
- **AC network islands** handled independently via connected component analysis
- **Newton-Raphson solvers** for both AC and DC power flow

### Applications

- Point-to-point HVDC transmission analysis
- Multi-terminal HVDC (MTDC) grid studies
- Offshore wind farm grid integration
- DC grid interconnection studies
- Hybrid AC/DC system planning

### Dependencies

- Python 3.9+
- NumPy
- SciPy
- Pandas
- PyPower

## Quick Start

### Basic Usage Pattern

```python
import acdcpf as pf

# 1. Create an empty network
net = pf.create_empty_network(name="My Network", s_base=100.0)

# 2. Add AC elements
b0 = pf.create_ac_bus(net, vr_kv=380.0, name="AC-Slack")
b1 = pf.create_ac_bus(net, vr_kv=380.0, name="AC-Load")
pf.create_ac_line(net, b0, b1, length_km=100, r_ohm_per_km=0.02, x_ohm_per_km=0.268)
pf.create_ac_gen(net, bus=b0, p_mw=0.0, v_pu=1.0)
pf.create_ac_load(net, bus=b1, p_mw=500.0, q_mvar=100.0)

# 3. Add DC elements
dc0 = pf.create_dc_bus(net, v_base=400.0, bus_type="vdc", v_dc_pu=1.0)
dc1 = pf.create_dc_bus(net, v_base=400.0, bus_type="p", v_dc_pu=1.0)
pf.create_dc_line(net, dc0, dc1, length_km=100, r_ohm_per_km=0.0114)

# 4. Add VSC converters
pf.create_vsc(net, ac_bus=b0, dc_bus=dc0, s_mva=600, control_mode="vdc_q", v_dc_pu=1.0)
pf.create_vsc(net, ac_bus=b1, dc_bus=dc1, s_mva=600, control_mode="p_q", p_mw=500.0)

# 5. Run power flow
converged = pf.run_pf(net, verbose=True)

# 6. Access results
print(net.res_ac_bus)
print(net.res_dc_bus)
print(net.res_vsc)
```

## Network Data Model

```python
from acdcpf import create_empty_network

net = create_empty_network(
```

### VSC Converters

The VSC (Voltage Source Converter) is the heart of AC/DC coupling. ACDCPF supports six control modes:

```python
idx = pf.create_vsc(
    net,                       # Network object
    ac_bus=0,                  # AC bus index
    dc_bus=0,                  # DC bus index
    s_mva=600.0,               # Rated apparent power (MVA)
    control_mode="p_q",        # Control mode (see table below)
    p_mw=500.0,                # Active power setpoint (MW)
    q_mvar=0.0,                # Reactive power setpoint (MVAr)
    v_ac_pu=1.0,               # AC voltage setpoint (pu)
    v_dc_pu=1.0,               # DC voltage setpoint (pu)
    droop_kv_per_mw=0.025,     # Droop constant (kV/MW)
    p_dc_set_mw=0.0,           # Droop P setpoint (MW)
    v_dc_set_pu=1.0,           # Droop Vdc setpoint (pu)
    loss_a=1.0,                # Constant loss term (MW)
    loss_b=0.0,                # Linear loss term (kV)
    loss_c=0.0,                # Quadratic loss term (Ohm)
    r_pu=0.01,                 # Converter resistance (pu)
    x_pu=0.10,                 # Converter reactance (pu)
    b_filter_pu=0.05,          # Filter susceptance (pu)
    in_service=True,           # Converter in service
    name="VSC1"                # Converter name
)
```

**Control Modes:**

| Mode | AC Control | DC Control | Use Case |
|------|------------|------------|----------|
| `"p_q"` | Constant P, Q | - | Standard power injection |
| `"p_vac"` | Constant P, Vac | - | Offshore wind (AC voltage control) |
| `"vdc_q"` | Constant Q | Vdc slack | DC voltage control |
| `"vdc_vac"` | Vac control | Vdc slack | Full voltage control |
| `"droop_q"` | Constant Q | P-Vdc droop | Autonomous power sharing |
| `"droop_vac"` | Vac control | P-Vdc droop | Vac + droop control |

**Power Convention:**
- Positive P_s (AC-side power) = **Rectifier** (AC → DC)
- Negative P_s = **Inverter** (DC → AC)

**Droop Control:**
The P-Vdc droop characteristic follows:
```
P = P0 - (1/k) * (V_dc - V0)
```
Where:
- `k` = droop constant (kV/MW), specified as `droop_kv_per_mw`
- `P0` = power setpoint, specified as `p_dc_set_mw`
- `V0` = voltage setpoint, specified as `v_dc_set_pu`

**Converter Losses:**
Losses are modeled as:
```
P_loss = a + b * |I_c| + c * |I_c|^2
```
Where `I_c` is the converter current magnitude.

### DC-DC Converters

DC-DC converters interconnect DC grids of different voltage levels.

```python
idx = pf.create_dcdc(
    net,                    # Network object
    from_bus=0,             # High-voltage bus index (V_m)
    to_bus=1,               # Low-voltage bus index (V_c)
    d_ratio=0.5,            # Voltage ratio D = V_c / V_m
    r_ohm=1.92,             # Series resistance (Ohm)
    g_us=0.39,              # Shunt conductance (uS)
    in_service=True,        # Converter in service
    name="DCDC1"            # Converter name
)
```

**Voltage Transformation:**
```
V_c = D * V_m
```

**Power Flow Equations:**
```
P_m = D^2 * G_s * V_m^2 - D * G_s * V_m * V_c + G_shunt * V_m
```

## Power Flow Analysis

### Running Power Flow

```python
from acdcpf.powerflow import run_pf

converged = run_pf(
    net,                   # Network object
    verbose=False,         # Print iteration details
    tol=1e-8,              # Convergence tolerance (MW)
    max_iter_outer=50,     # Maximum outer iterations
    max_iter_inner=30      # Maximum inner (NR) iterations
)
```

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `net` | Network | - | Network object (required) |
| `verbose` | bool | False | Print convergence details |
| `tol` | float | 1e-8 | Power mismatch tolerance (MW) |
| `max_iter_outer` | int | 50 | Max sequential iterations |
| `max_iter_inner` | int | 30 | Max Newton-Raphson iterations |

**Return Value:**
- `True` if converged within tolerance
- `False` if maximum iterations reached

### Convergence

The power flow is considered converged when:
```
max|P_s^(k) - P_s^(k-1)| < tolerance
```

Where `P_s` is the AC-side power of all VSC converters.

Typical convergence:
- 3-10 outer iterations for well-posed problems
- <5 inner NR iterations per outer iteration

## Algorithms and Theory

### Sequential AC/DC Power Flow Method

ACDCPF implements the sequential AC/DC power flow algorithm, which iterates between AC and DC subsystems until convergence.

**Algorithm Flowchart:**

```
┌─────────────────────────────────────────┐
│           Initialize                    │
│  - Set initial VSC powers (P_s, Q_s)    │
│  - Set initial DC voltages (V_dc)       │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         AC Power Flow                   │
│  - Newton-Raphson via PyPower           │
│  - Get V_ac at all buses                │
│  - Handle AC islands independently      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│       Converter Calculations            │
│  - Calculate V_f (filter voltage)       │
│  - Calculate I_tf, I_c (currents)       │
│  - Calculate P_loss (losses)            │
│  - Calculate P_dc from AC solution      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         DC Power Flow                   │
│  - Newton-Raphson on DC network         │
│  - Get V_dc at all buses                │
│  - Update slack/droop converter powers  │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│     Update Slack/Droop VSC Powers       │
│  - Backward solver: P_dc → P_s          │
│  - Update AC-side power injections      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         Check Convergence               │
│  max|P_s^(k) - P_s^(k-1)| < tol ?       │
└───────────────────┬─────────────────────┘
                    │
         ┌──────────┴──────────┐
         │ No                  │ Yes
         ▼                     ▼
    [Next Iteration]      [Converged]
```

### VSC Converter Model

The VSC is modeled with three main components:

1. **AC-side transformer** with impedance Z_tf = R + jX
2. **Filter** with shunt susceptance B_f
3. **Power electronic converter** with losses

**Equivalent Circuit:**

```
AC Bus (V_s)                                    DC Bus (V_dc)
    │                                               │
    │    ┌───────────┐                              │
    ├────┤  Z_tf     ├────┬───────[Converter]──────┤
    │    │ R + jX    │    │         P_loss          │
    │    └───────────┘    │                         │
    │                     │                         │
    │               ┌─────┴─────┐                   │
    │               │   jB_f    │                   │
    │               │  (Filter) │                   │
    │               └───────────┘                   │
    │                                               │
```

**Key Equations (Beerten Equations 13.11-13.23):**

1. **Transformer current:**
```
I_tf = conj(S_s / V_s)
```

2. **Filter voltage:**
```
V_f = V_s + I_tf * Z_tf
```

3. **Filter-side power:**
```
S_sf = V_f * conj(I_tf)
```

4. **Filter reactive power:**
```
Q_f = -B_f * |V_f|^2
```

5. **Converter-side power:**
```
S_cf = S_sf + j*Q_f
```

6. **Converter current:**
```
I_c = conj(S_cf / V_f)
```

7. **Converter losses:**
```
P_loss = a + b*|I_c| + c*|I_c|^2
```

8. **DC-side power:**
```
P_dc = Re(S_cf) - P_loss
```

### DC Network Model

The DC network is modeled as a purely resistive network using conductance:

**Conductance Matrix (G_dc):**

For a DC line between buses i and j with resistance R:
```
G_ij = 1/R
G_ii += G_ij
G_jj += G_ij
```

**Power Balance Equation:**

For each DC bus i:
```
P_i = pol * V_i * Σ_j(G_ij * V_j)
```

Where `pol` is the number of poles (1 or 2).

### DC Bus Types

**1. Vdc Slack Bus (`bus_type="vdc"`)**
- Voltage is specified: V_i = V_spec
- Power is calculated from the DC network solution
- At least one Vdc slack required per DC grid

**2. P-Controlled Bus (`bus_type="p"`)**
- Power is specified by the connected VSC
- Voltage is calculated via Newton-Raphson
- Mismatch: `P_spec - P_calc = 0`

**3. Droop-Controlled Bus (`bus_type="droop"`)**
- Power follows the droop characteristic
- Voltage is calculated via Newton-Raphson
- Droop equation: `P = P0 - (1/k) * (V - V0)`
- Mismatch: `P_droop - P_calc = 0`

### Newton-Raphson for DC Power Flow

**Jacobian Matrix:**

For P-controlled buses:
```
J_ii = -P_calc_i / V_i - pol * V_i * G_ii
J_ij = -pol * V_i * G_ij
```

For droop-controlled buses:
```
J_ii = -P_calc_i / V_i - pol * V_i * G_ii - 1/k_i
J_ij = -pol * V_i * G_ij
```

**Update:**
```
ΔV = -J^(-1) * mismatch
V^(k+1) = V^(k) + ΔV
```

### Backward Converter Solver

For Vdc-slack and droop VSCs, the DC solution determines P_dc. The backward solver finds the corresponding AC-side power P_s:

**Algorithm:**
```
Given: P_dc from DC solution, V_s from AC solution

Initialize: p_s = P_dc (approximation)
Iterate until convergence:
    S_s = p_s + j*q_s
    I_tf = conj(S_s / V_s)
    V_f = V_s + I_tf * Z_tf
    S_cf = V_f * conj(I_tf) + j*Q_f
    I_c = conj(S_cf / V_f)
    P_loss = a + b*|I_c| + c*|I_c|^2
    p_s_new = (P_dc + P_loss + R_tf*|I_tf|^2) / S_base
    Check: |p_s_new - p_s| < tolerance
    p_s = p_s_new
```

```python
"""
Example 1: Simple 2-Terminal HVDC System

This example creates a point-to-point HVDC link connecting two AC buses.
One converter controls DC voltage, the other controls power.
"""
import acdcpf as pf
from acdcpf.powerflow import run_pf

# Create network
net = pf.create_empty_network(name="Simple HVDC", s_base=100.0, f_hz=50.0)

# === AC System ===
# Add AC buses
ac_slack = pf.create_ac_bus(net, vr_kv=400.0, name="AC-Slack")
ac_load = pf.create_ac_bus(net, vr_kv=400.0, name="AC-Load")

# Add AC transmission line (100 km)
pf.create_ac_line(net,
    from_bus=ac_slack, to_bus=ac_load,
    length_km=100.0,
    r_ohm_per_km=0.025,
    x_ohm_per_km=0.30,
    b_us_per_km=4.0,
    name="AC-Line-1"
)

# Add generator at slack bus
pf.create_ac_gen(net, bus=ac_slack, p_mw=0.0, v_pu=1.02, name="Slack-Gen")

# Add load at load bus
pf.create_ac_load(net, bus=ac_load, p_mw=800.0, q_mvar=200.0, name="Main-Load")

# === DC System ===
# Add DC buses
dc_rect = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="p",
                           v_dc_pu=1.0, name="DC-Rectifier")
dc_inv = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="vdc",
                          v_dc_pu=1.0, name="DC-Inverter")

# Add DC line (200 km submarine cable)
pf.create_dc_line(net,
    from_bus=dc_rect, to_bus=dc_inv,
    length_km=200.0,
    r_ohm_per_km=0.01,
    name="DC-Cable"
)

# === VSC Converters ===
# Rectifier: P control at 600 MW
pf.create_vsc(net,
    ac_bus=ac_slack, dc_bus=dc_rect,
    s_mva=800.0,
    control_mode="p_q",
    p_mw=600.0,
    q_mvar=0.0,
    loss_a=1.0,
    r_pu=0.01,
    x_pu=0.12,
    name="VSC-Rectifier"
)

# Inverter: Vdc control at 1.0 pu
pf.create_vsc(net,
    ac_bus=ac_load, dc_bus=dc_inv,
    s_mva=800.0,
    control_mode="vdc_q",
    v_dc_pu=1.0,
    q_mvar=50.0,  # Provide some reactive power support
    loss_a=1.0,
    r_pu=0.01,
    x_pu=0.12,
    name="VSC-Inverter"
)

# === Run Power Flow ===
print("Running power flow...")
converged = run_pf(net, verbose=True, tol=1e-6)

# === Display Results ===
print(f"\nConverged: {converged}")
print("\n=== AC Bus Results ===")
print(net.res_ac_bus)
print("\n=== DC Bus Results ===")
print(net.res_dc_bus)
print("\n=== VSC Results ===")
print(net.res_vsc)
print("\n=== DC Line Results ===")
print(net.res_dc_line)
```

### Example 2: Multi-Terminal DC with Droop Control

```python
"""
Example 2: 3-Terminal DC Grid with Droop Control

This example demonstrates:
- Multiple converters sharing power via droop control
- One Vdc-slack converter for voltage reference
- Two droop converters for power sharing
"""
import acdcpf as pf
from acdcpf.powerflow import run_pf

net = pf.create_empty_network(name="3-Terminal MTDC", s_base=100.0)

# === AC Systems (3 independent grids) ===
# Grid A: Generation
ac_a = pf.create_ac_bus(net, vr_kv=400.0, name="AC-A")
pf.create_ac_gen(net, bus=ac_a, p_mw=0.0, v_pu=1.0, name="Gen-A")
pf.create_ac_load(net, bus=ac_a, p_mw=-500.0, q_mvar=0.0, name="Gen-A-equiv")

# Grid B: Load center 1
ac_b = pf.create_ac_bus(net, vr_kv=400.0, name="AC-B")
pf.create_ac_gen(net, bus=ac_b, p_mw=0.0, v_pu=1.0, name="Gen-B")
pf.create_ac_load(net, bus=ac_b, p_mw=300.0, q_mvar=100.0, name="Load-B")

# Grid C: Load center 2
ac_c = pf.create_ac_bus(net, vr_kv=400.0, name="AC-C")
pf.create_ac_gen(net, bus=ac_c, p_mw=0.0, v_pu=1.0, name="Gen-C")
pf.create_ac_load(net, bus=ac_c, p_mw=200.0, q_mvar=50.0, name="Load-C")

# === DC Grid ===
# DC Bus A: Connected to generation, Vdc slack
dc_a = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="vdc",
                        v_dc_pu=1.0, name="DC-A")
# DC Bus B: Droop controlled
dc_b = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="droop",
                        v_dc_pu=1.0, name="DC-B")
# DC Bus C: Droop controlled
dc_c = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="droop",
                        v_dc_pu=1.0, name="DC-C")

# DC lines (star topology: A is hub)
pf.create_dc_line(net, dc_a, dc_b, length_km=150.0, r_ohm_per_km=0.012, name="DC-A-B")
pf.create_dc_line(net, dc_a, dc_c, length_km=200.0, r_ohm_per_km=0.012, name="DC-A-C")

# === VSC Converters ===
# VSC-A: Vdc control (slack)
pf.create_vsc(net, ac_bus=ac_a, dc_bus=dc_a, s_mva=600.0,
              control_mode="vdc_vac", v_dc_pu=1.0, v_ac_pu=1.0,
              r_pu=0.01, x_pu=0.1, loss_a=0.6, name="VSC-A")

# VSC-B: Droop control (P0=-200 MW, k=0.02 kV/MW)
pf.create_vsc(net, ac_bus=ac_b, dc_bus=dc_b, s_mva=400.0,
              control_mode="droop_vac", v_ac_pu=1.0,
              droop_kv_per_mw=0.02,
              p_dc_set_mw=-200.0,  # Target inverter mode
              v_dc_set_pu=1.0,
              r_pu=0.01, x_pu=0.1, loss_a=0.4, name="VSC-B")

# VSC-C: Droop control (P0=-150 MW, k=0.02 kV/MW)
pf.create_vsc(net, ac_bus=ac_c, dc_bus=dc_c, s_mva=300.0,
              control_mode="droop_vac", v_ac_pu=1.0,
              droop_kv_per_mw=0.02,
              p_dc_set_mw=-150.0,
              v_dc_set_pu=1.0,
              r_pu=0.01, x_pu=0.1, loss_a=0.3, name="VSC-C")

# === Run and analyze ===
converged = run_pf(net, verbose=True)

print(f"\nConverged: {converged}")
print("\n=== VSC Power Sharing ===")
for idx, row in net.vsc.iterrows():
    p_ac = net.res_vsc.loc[idx, 'p_ac_mw']
    p_dc = net.res_vsc.loc[idx, 'p_dc_mw']
    mode = "Rectifier" if p_ac > 0 else "Inverter"
    print(f"{row['name']}: P_ac={p_ac:.1f} MW, P_dc={p_dc:.1f} MW ({mode})")

print("\n=== DC Voltages ===")
for idx, row in net.dc_bus.iterrows():
    v_pu = net.res_dc_bus.loc[idx, 'v_dc_pu']
    print(f"{row['name']}: V_dc={v_pu:.4f} pu")
```

### Example 3: Offshore Wind Integration

```python
"""
Example 3: Offshore Wind Farm Integration

This example models an offshore wind farm connected to onshore grid via HVDC.
The offshore converter controls AC voltage for the wind farm.
"""
import acdcpf as pf
from acdcpf.powerflow import run_pf

net = pf.create_empty_network(name="Offshore Wind HVDC", s_base=100.0)

# === Onshore AC System (380 kV) ===
ac_onshore = pf.create_ac_bus(net, vr_kv=380.0, name="Onshore-PCC")
pf.create_ac_gen(net, bus=ac_onshore, p_mw=0.0, v_pu=1.0, name="Grid-Equiv")
pf.create_ac_load(net, bus=ac_onshore, p_mw=1000.0, q_mvar=300.0, name="Onshore-Load")

# === Offshore AC System (66 kV collection) ===
ac_offshore = pf.create_ac_bus(net, vr_kv=66.0, name="Offshore-Collection")
# Wind farm as negative load (generation)
pf.create_ac_load(net, bus=ac_offshore, p_mw=-500.0, q_mvar=0.0, name="Wind-Farm")

# === DC System (+/-320 kV submarine) ===
dc_offshore = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="p",
                               v_dc_pu=1.0, name="DC-Offshore")
dc_onshore = pf.create_dc_bus(net, v_base=320.0, dc_grid=0, bus_type="vdc",
                              v_dc_pu=1.0, name="DC-Onshore")

# Submarine DC cable (100 km)
pf.create_dc_line(net, dc_offshore, dc_onshore, length_km=100.0,
                  r_ohm_per_km=0.01, name="Submarine-Cable")

# === VSC Converters ===
# Offshore: P-Vac control (AC voltage for wind farm, P follows wind)
# P is 0 because wind injects into the bus, VSC just passes it through
pf.create_vsc(net, ac_bus=ac_offshore, dc_bus=dc_offshore, s_mva=600.0,
              control_mode="p_vac",
              p_mw=0.0,  # Wind injects, VSC balances
              v_ac_pu=1.0,
              r_pu=0.01, x_pu=0.15,
              loss_a=0.6, name="VSC-Offshore")

# Onshore: Vdc-Vac control
pf.create_vsc(net, ac_bus=ac_onshore, dc_bus=dc_onshore, s_mva=600.0,
              control_mode="vdc_vac",
              v_dc_pu=1.0,
              v_ac_pu=1.0,
              r_pu=0.01, x_pu=0.15,
              loss_a=0.6, name="VSC-Onshore")

# === Run power flow ===
converged = run_pf(net, verbose=True)

print(f"\nConverged: {converged}")
print("\n=== Power Transfer ===")
offshore_p = net.res_vsc.loc[0, 'p_ac_mw']
onshore_p = net.res_vsc.loc[1, 'p_ac_mw']
losses = offshore_p + onshore_p  # Should be negative (rectifier + inverter)
cable_loss = net.res_dc_line.loc[0, 'p_loss_mw']

print(f"Offshore VSC: {offshore_p:.1f} MW (rectifying wind power)")
print(f"Onshore VSC: {onshore_p:.1f} MW (delivering to grid)")
print(f"Cable losses: {cable_loss:.2f} MW")
print(f"VSC losses: {abs(offshore_p) - abs(onshore_p) - cable_loss:.2f} MW")
```

## Appendix A: Sign Conventions

### Power Convention

| Element | Positive Power | Negative Power |
|---------|----------------|----------------|
| AC Load | Consumption | Generation |
| AC Generator | Generation | N/A |
| VSC (P_s) | Rectifier (AC→DC) | Inverter (DC→AC) |
| VSC (P_dc) | Into DC grid | Out of DC grid |

### Voltage Convention

- AC voltages are complex phasors: `V = |V| * exp(j*theta)`
- DC voltages are real positive values
- Per-unit system based on `s_base` and element rated voltages

## Appendix B: Troubleshooting

### Common Issues

**1. Power flow does not converge**
- Check that each DC grid has exactly one Vdc slack bus
- Ensure load/generation balance is reasonable
- Try increasing `max_iter_outer`
- Check VSC ratings are sufficient for required power transfer

**2. Unexpected power flow direction**
- Remember: positive P_s means rectifier (AC→DC)
- Check droop setpoints (P0 and V0)
- Verify load/generation signs

**3. DC voltage out of range**
- Adjust droop constants for better voltage regulation
- Check DC line resistances
- Consider adding reactive power support at AC buses
