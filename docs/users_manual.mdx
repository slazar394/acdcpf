## Introduction

ACDCPF is a Python library for power flow analysis in hybrid AC/DC networks. It implements the **sequential AC/DC power flow method** based on the algorithms described in Beerten's work on VSC-HVDC systems.

Power flow analysis computes the steady-state voltages, currents, and power flows throughout an electrical network given its topology, line parameters, loads, and generators. In hybrid AC/DC systems, AC and DC sub-networks are coupled through Voltage Source Converters (VSCs), which makes the problem harder — the AC and DC solutions depend on each other. ACDCPF handles this by solving each side iteratively until the whole system converges.

### Key Features

- **Multi-terminal VSC-HVDC systems** with various converter control modes
- **DC-DC converters** for interconnecting DC grids of different voltage levels
- **Multiple independent DC grids** with proper voltage control coordination
- **Six VSC control modes**: P-Q, P-Vac, Vdc-Q, Vdc-Vac, Droop-Q, Droop-Vac
- **DC loads and generators** for modeling DC-side equipment
- **AC network islands** handled independently via connected component analysis
- **Newton-Raphson solvers** for both AC and DC power flow

### Applications

- Point-to-point HVDC transmission analysis
- Multi-terminal HVDC (MTDC) grid studies
- Offshore wind farm grid integration
- DC grid interconnection studies
- Hybrid AC/DC system planning

### Dependencies

- Python 3.9+
- NumPy
- SciPy
- Pandas
- PyPower

## Quick Start

### Basic Usage Pattern

```python
import acdcpf as pf

# 1. Create an empty network
net = pf.create_empty_network(name="My Network", s_base=100.0)

# 2. Add AC elements
b0 = pf.create_ac_bus(net, vr_kv=380.0, name="AC-Slack")
b1 = pf.create_ac_bus(net, vr_kv=380.0, name="AC-Load")
pf.create_ac_line(net, b0, b1, length_km=100, r_ohm_per_km=0.02, x_ohm_per_km=0.268)
pf.create_ac_gen(net, bus=b0, p_mw=0.0, v_pu=1.0)
pf.create_ac_load(net, bus=b1, p_mw=500.0, q_mvar=100.0)

# 3. Add DC elements
dc0 = pf.create_dc_bus(net, v_base=400.0, bus_type="vdc", v_dc_pu=1.0)
dc1 = pf.create_dc_bus(net, v_base=400.0, bus_type="p", v_dc_pu=1.0)
pf.create_dc_line(net, dc0, dc1, length_km=100, r_ohm_per_km=0.0114)

# 4. Add VSC converters
pf.create_vsc(net, ac_bus=b0, dc_bus=dc0, s_mva=600, control_mode="vdc_q", v_dc_pu=1.0)
pf.create_vsc(net, ac_bus=b1, dc_bus=dc1, s_mva=600, control_mode="p_q", p_mw=500.0)

# 5. Run power flow
converged = pf.run_pf(net, verbose=True)

# 6. Access results
print(net.res_ac_bus)
print(net.res_dc_bus)
print(net.res_vsc)
```

## Network Data Model

### Network Object

The `Network` class is the central data container that holds all network elements as pandas DataFrames.

```python
from acdcpf import create_empty_network

net = create_empty_network(
    name="My Network",    # Network name
    s_base=100.0,         # System base power (MVA)
    f_hz=50.0,            # System frequency (Hz)
    pol=2                 # DC poles (2 for bipole/symmetric monopole)
)
```

### Network Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `name` | str | Network name |
| `s_base` | float | Base power (MVA), default 100.0 |
| `f_hz` | float | System frequency (Hz), default 50.0 |
| `pol` | int | DC poles (1=asymmetric monopole, 2=bipole/symmetric monopole) |
| `ac_bus` | DataFrame | AC bus data |
| `ac_line` | DataFrame | AC transmission line data |
| `ac_load` | DataFrame | AC load data |
| `ac_gen` | DataFrame | AC generator data |
| `dc_bus` | DataFrame | DC bus data |
| `dc_line` | DataFrame | DC line data |
| `dc_load` | DataFrame | DC load data |
| `dc_gen` | DataFrame | DC generator data |

| `dcdc` | DataFrame | DC-DC converter data |

- The `pol` parameter in `create_empty_network` multiplies DC power by the number of poles: `P = pol * V * G * V`

### VSC Converters

The VSC (Voltage Source Converter) is the heart of AC/DC coupling. ACDCPF supports six control modes:

```python
idx = pf.create_vsc(
    net,                       # Network object
    ac_bus=0,                  # AC bus index
    dc_bus=0,                  # DC bus index
    s_mva=600.0,               # Rated apparent power (MVA)
    control_mode="p_q",        # Control mode (see table below)
    p_mw=500.0,                # Active power setpoint (MW)
    q_mvar=0.0,                # Reactive power setpoint (MVAr)
    v_ac_pu=1.0,               # AC voltage setpoint (pu)
    v_dc_pu=1.0,               # DC voltage setpoint (pu)
    droop_kv_per_mw=0.025,     # Droop constant (kV/MW)
    p_dc_set_mw=0.0,           # Droop P setpoint (MW)
    v_dc_set_pu=1.0,           # Droop Vdc setpoint (pu)
    loss_a=1.0,                # Constant loss term (MW)
    loss_b=0.0,                # Linear loss term (kV)
    loss_c=0.0,                # Quadratic loss coefficient, rectifier (Ohm)
    loss_c_inv=0.0,            # Quadratic loss coefficient, inverter (Ohm)
    r_tf_pu=0.01,              # Transformer resistance (pu)
    x_tf_pu=0.10,              # Transformer reactance (pu)
    r_c_pu=0.0,                # Phase reactor resistance (pu)
    x_c_pu=0.0,                # Phase reactor reactance (pu)
    b_filter_pu=0.05,          # Filter susceptance (pu)
    loss_base_kv=None,         # AC base voltage for loss calc (kV)
    in_service=True,           # Converter in service
    name="VSC1"                # Converter name
)
```

**Control Modes:**

| Mode | AC Control | DC Control | Use Case |
|------|------------|------------|----------|
| `"p_q"` | Constant P, Q | - | Standard power injection |
| `"p_vac"` | Constant P, Vac | - | Offshore wind (AC voltage control) |
| `"vdc_q"` | Constant Q | Vdc slack | DC voltage control |
| `"vdc_vac"` | Vac control | Vdc slack | Full voltage control |
| `"droop_q"` | Constant Q | P-Vdc droop | Autonomous power sharing |
| `"droop_vac"` | Vac control | P-Vdc droop | Vac + droop control |

**Power Convention:**
- Positive P_s (AC-side power) = **Rectifier** (AC → DC)
- Negative P_s = **Inverter** (DC → AC)

**Droop Control:**
The P-Vdc droop characteristic follows:
```
P = P0 - (1/k) * (V_dc - V0)
```
Where:
- `k` = droop constant (kV/MW), specified as `droop_kv_per_mw`
- `P0` = power setpoint, specified as `p_dc_set_mw`
- `V0` = voltage setpoint, specified as `v_dc_set_pu`

**Converter Losses:**
Losses are modeled as:
```
P_loss = a + b * |I_c| + c * |I_c|^2
```
Where `I_c` is the converter current magnitude and `c` is selected based on operating mode:
- `loss_c` (maps to MatACDC `LossCrec`) is used when the converter operates as **inverter** in acdcpf convention (P_c < 0, DC→AC)
- `loss_c_inv` (maps to MatACDC `LossCinv`) is used when the converter operates as **rectifier** in acdcpf convention (P_c > 0, AC→DC)

If `loss_c_inv` is not specified, it defaults to `loss_c`.

**Loss Base Voltage:**
The `loss_base_kv` parameter specifies the AC base voltage used for converting loss coefficients from physical to per-unit. If omitted, the rated voltage of the connected AC bus is used. This matches MatACDC's `basekVac` field, which is only used for loss coefficient conversion (not impedance scaling).

**Vac/Generator Conflict:**
When a VSC with AC voltage control (`p_vac`, `vdc_vac`, `droop_vac`) is connected to a bus that already has an AC generator controlling voltage (PV bus), the VSC's Vac control is automatically removed and its reactive power is set to zero. This prevents conflicting voltage setpoints and matches MatACDC's behavior.

**DataFrame columns (`net.vsc`):**

| Column | Type | Unit | Description |
|--------|------|------|-------------|
| `name` | str | - | Converter name |
| `ac_bus` | int | - | AC bus index |
| `dc_bus` | int | - | DC bus index |
| `s_mva` | float | MVA | Rated power |
| `control_mode` | str | - | Control mode |
| `p_mw` | float | MW | P setpoint |
| `q_mvar` | float | MVAr | Q setpoint |
| `v_ac_pu` | float | pu | AC voltage setpoint |
| `v_dc_pu` | float | pu | DC voltage setpoint |
| `droop_kv_per_mw` | float | kV/MW | Droop constant |
| `p_dc_set_mw` | float | MW | Droop P setpoint |
| `v_dc_set_pu` | float | pu | Droop V setpoint |
| `loss_a` | float | MW | Constant loss |
| `loss_b` | float | kV | Linear loss coefficient |
| `loss_c` | float | Ohm | Quadratic loss coefficient (rectifier) |
| `loss_c_inv` | float | Ohm | Quadratic loss coefficient (inverter) |
| `r_tf_pu` | float | pu | Transformer resistance |

### DC-DC Converters

DC-DC converters interconnect DC grids of different voltage levels.

```python
idx = pf.create_dcdc(
    net,                    # Network object
    from_bus=0,             # High-voltage bus index (V_m)
    to_bus=1,               # Low-voltage bus index (V_c)
    d_ratio=0.5,            # Voltage ratio D = V_c / V_m
    r_ohm=1.92,             # Series resistance (Ohm)
    g_us=0.39,              # Shunt conductance (uS)
    in_service=True,        # Converter in service
    name="DCDC1"            # Converter name
)
```

**Voltage Transformation:**
```
V_c = D * V_m
```

**Power Flow Equations:**
```
P_m = D^2 * G_s * V_m^2 - D * G_s * V_m * V_c + G_shunt * V_m
```

**DataFrame columns (`net.dcdc`):**

| Column | Type | Unit | Description |
|--------|------|------|-------------|
| `name` | str | - | Converter name |
| `from_bus` | int | - | High-voltage bus (V_m) |
| `to_bus` | int | - | Low-voltage bus (V_c) |
| `d_ratio` | float | - | Voltage ratio |
| `r_ohm` | float | Ohm | Series resistance |
| `g_us` | float | uS | Shunt conductance |
| `in_service` | bool | - | Converter status |

## Power Flow Analysis

### Running Power Flow

```python
converged = pf.run_pf(
    net,                   # Network object
    verbose=False,         # Print iteration details
    tol=1e-8,              # Convergence tolerance (MW)
    max_iter_outer=50,     # Maximum outer iterations
    max_iter_inner=30      # Maximum inner (NR) iterations
)
```

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `net` | Network | - | Network object (required) |
| `verbose` | bool | False | Print convergence details |
| `tol` | float | 1e-8 | Power mismatch tolerance (MW) |
| `max_iter_outer` | int | 50 | Max sequential iterations |
| `max_iter_inner` | int | 30 | Max Newton-Raphson iterations |

**Return Value:**
- `True` if converged within tolerance
- `False` if maximum iterations reached

### Convergence

The power flow is considered converged when:
```
max|P_s^(k) - P_s^(k-1)| < tolerance
```

Where `P_s` is the AC-side power of all VSC converters.

Typical convergence:
- 3-10 outer iterations for well-posed problems
- <5 inner NR iterations per outer iteration

## Algorithms and Theory

### Sequential AC/DC Power Flow Method

ACDCPF implements the sequential AC/DC power flow algorithm, which iterates between AC and DC subsystems until convergence.

**Algorithm Flowchart:**

```
┌─────────────────────────────────────────┐
│           Initialize                    │
│  - Set initial VSC powers (P_s, Q_s)    │
│  - Set initial DC voltages (V_dc)       │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         AC Power Flow                   │
│  - Newton-Raphson via PyPower           │
│  - Get V_ac at all buses                │
│  - Handle AC islands independently      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│       Converter Calculations            │
│  - Calculate V_f (filter voltage)       │
│  - Calculate I_tf, I_c (currents)       │
│  - Calculate P_loss (losses)            │
│  - Calculate P_dc from AC solution      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         DC Power Flow                   │
│  - Newton-Raphson on DC network         │
│  - Get V_dc at all buses                │
│  - Update slack/droop converter powers  │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│     Update Slack/Droop VSC Powers       │
│  - Backward solver: P_dc → P_s          │
│  - Update AC-side power injections      │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         Check Convergence               │
│  max|P_s^(k) - P_s^(k-1)| < tol ?       │
└───────────────────┬─────────────────────┘
                    │
         ┌──────────┴──────────┐
         │ No                  │ Yes
         ▼                     ▼
    [Next Iteration]      [Converged]
```

### VSC Converter Model

The VSC is modeled with four main components:

1. **AC-side transformer** with impedance Z_tf = R_tf + jX_tf
2. **Filter** with shunt susceptance B_f
3. **Phase reactor** with impedance Z_c = R_c + jX_c
4. **Power electronic converter** with losses

**Equivalent Circuit:**

```
AC Bus (V_s)                                             DC Bus (V_dc)
    │                                                        │
    │    ┌───────────┐         ┌───────────┐                │
    ├────┤  Z_tf     ├────┬────┤  Z_c      ├──[Converter]──┤
    │    │ Rtf + jXtf│    │    │ Rc + jXc  │    P_loss      │
    │    └───────────┘    │    └───────────┘                 │
    │                     │                                  │
    │               ┌─────┴─────┐                            │
    │               │   jB_f    │                            │
    │               │  (Filter) │                            │
    │               └───────────┘                            │
    │                                                        │
```

**Note:** The phase reactor (Z_c) is optional. When `r_c_pu=0` and `x_c_pu=0`, the model reduces to a single-impedance transformer model.

**Key Equations (Beerten Equations 13.11-13.23):**

1. **Transformer current:**
```
I_tf = conj(S_s / V_s)
```

2. **Filter voltage:**
```
V_f = V_s + I_tf * Z_tf       (where Z_tf = R_tf + jX_tf)
```

3. **Filter-side power:**
```
S_sf = V_f * conj(I_tf)
```

4. **Filter reactive power:**
```
Q_f = -B_f * |V_f|^2
```

5. **Converter-side power:**
```
S_cf = S_sf + j*Q_f
```

6. **Converter current:**
```
I_c = conj(S_cf / V_f)
```

7. **Converter losses (sign-dependent):**
```
P_loss = a + b*|I_c| + c*|I_c|^2
```
Where `c = loss_c_inv` if rectifying (P_c > 0), `c = loss_c` if inverting (P_c < 0).

8. **DC-side power:**
```
P_dc = Re(S_cf) - P_loss
```

### DC Network Model

The DC network is modeled as a purely resistive network using conductance:

**Conductance Matrix (G_dc):**

For a DC line between buses i and j with resistance R:
```
G_ij = 1/R
G_ii += G_ij
G_jj += G_ij
```

**Power Balance Equation:**

For each DC bus i:
```
P_i = pol * V_i * Σ_j(G_ij * V_j)
```

Where `pol` is the number of poles (1 or 2).

### DC Bus Types

**1. Vdc Slack Bus (`bus_type="vdc"`)**
- Voltage is specified: V_i = V_spec
- Power is calculated from the DC network solution
- At least one Vdc slack required per DC grid

**2. P-Controlled Bus (`bus_type="p"`)**
- Power is specified by the connected VSC
- Voltage is calculated via Newton-Raphson
- Mismatch: `P_spec - P_calc = 0`

**3. Droop-Controlled Bus (`bus_type="droop"`)**
- Power follows the droop characteristic
- Voltage is calculated via Newton-Raphson
- Droop equation: `P = P0 - (1/k) * (V - V0)`
- Mismatch: `P_droop - P_calc = 0`

### Newton-Raphson for DC Power Flow

**Jacobian Matrix:**

For P-controlled buses:
```
J_ii = -P_calc_i / V_i - pol * V_i * G_ii
J_ij = -pol * V_i * G_ij
```

For droop-controlled buses:
```
J_ii = -P_calc_i / V_i - pol * V_i * G_ii - 1/k_i
J_ij = -pol * V_i * G_ij
```

**Update:**
```
ΔV = -J^(-1) * mismatch
V^(k+1) = V^(k) + ΔV
```

### Backward Converter Solver

For Vdc-slack and droop VSCs, the DC solution determines P_dc. The backward solver finds the corresponding AC-side power P_s:

**Algorithm:**
```
Given: P_dc from DC solution, V_s from AC solution

Initialize: p_s = P_dc (approximation)
Iterate until convergence:
    S_s = p_s + j*q_s
    I_tf = conj(S_s / V_s)
    V_f = V_s + I_tf * Z_tf
    S_cf = V_f * conj(I_tf) + j*Q_f
    I_c = conj(S_cf / V_f)
    P_loss = a + b*|I_c| + c*|I_c|^2
    p_s_new = P_dc + P_loss + R_tf*|I_tf|^2
    Check: |p_s_new - p_s| < tolerance
    p_s = p_s_new
```

### Example 2: Building a Network from Scratch

This example builds the 5-bus Stagg system with a 3-terminal MTDC grid and Vdc slack control. See [`examples/build_network.py`](../examples/build_network.py) for the fully commented version.

```python
import acdcpf as pf

net = pf.create_empty_network(name="Case5 Stagg MTDC Slack", s_base=100.0, pol=2)

base_kv = 345.0
z_base = base_kv**2 / net.s_base
y_base = 1.0 / z_base

# AC buses
b1 = pf.create_ac_bus(net, vr_kv=base_kv, name="Bus 1")
b2 = pf.create_ac_bus(net, vr_kv=base_kv, name="Bus 2")
b3 = pf.create_ac_bus(net, vr_kv=base_kv, name="Bus 3")
b4 = pf.create_ac_bus(net, vr_kv=base_kv, name="Bus 4")
b5 = pf.create_ac_bus(net, vr_kv=base_kv, name="Bus 5")

# AC loads
pf.create_ac_load(net, bus=b2, p_mw=20.0,  q_mvar=10.0)
pf.create_ac_load(net, bus=b3, p_mw=45.0,  q_mvar=15.0)
pf.create_ac_load(net, bus=b4, p_mw=40.0,  q_mvar=5.0)
pf.create_ac_load(net, bus=b5, p_mw=60.0,  q_mvar=10.0)

# AC generators (v_pu makes them PV buses; first PV = slack)
pf.create_ac_gen(net, bus=b1, p_mw=0.0,  v_pu=1.06)
pf.create_ac_gen(net, bus=b2, p_mw=40.0, v_pu=1.00)

# AC lines (using length_km=1.0 so per-km values = total values)
for fbus, tbus, r, x, b in [
    (b1,b2,0.02,0.06,0.06), (b1,b3,0.08,0.24,0.05), (b2,b3,0.06,0.18,0.04),
    (b2,b4,0.06,0.18,0.04), (b2,b5,0.04,0.12,0.03), (b3,b4,0.01,0.03,0.02),
    (b4,b5,0.08,0.24,0.05),
]:
    pf.create_ac_line(net, fbus, tbus, length_km=1.0,
                      r_ohm_per_km=r*z_base, x_ohm_per_km=x*z_base,
                      b_us_per_km=b*y_base*1e6)

# DC buses (one "vdc" slack per DC grid, rest are "p")
dc1 = pf.create_dc_bus(net, v_base=base_kv, dc_grid=1, bus_type="p",   v_dc_pu=1.0)
dc2 = pf.create_dc_bus(net, v_base=base_kv, dc_grid=1, bus_type="vdc", v_dc_pu=1.0)
dc3 = pf.create_dc_bus(net, v_base=base_kv, dc_grid=1, bus_type="p",   v_dc_pu=1.0)

# DC lines (resistive only)
for fbus, tbus, r_pu in [(dc1,dc2,0.052), (dc2,dc3,0.052), (dc1,dc3,0.073)]:
    pf.create_dc_line(net, fbus, tbus, length_km=1.0, r_ohm_per_km=r_pu*z_base)

# VSC converters
# p_mw > 0 = rectifier (AC->DC), p_mw < 0 = inverter (DC->AC)
conv_params = dict(s_mva=120.0, r_tf_pu=0.0015, x_tf_pu=0.1121,
                   r_c_pu=0.0001, x_c_pu=0.16428, b_filter_pu=0.0887,
                   loss_a=1.103, loss_b=0.887, loss_c=2.885, loss_c_inv=4.371)

pf.create_vsc(net, ac_bus=b2, dc_bus=dc1, control_mode="p_q",
              p_mw=60.0, q_mvar=40.0, **conv_params)
pf.create_vsc(net, ac_bus=b3, dc_bus=dc2, control_mode="vdc_vac",
              v_dc_pu=1.0, v_ac_pu=1.0, **conv_params)
pf.create_vsc(net, ac_bus=b5, dc_bus=dc3, control_mode="p_q",
              p_mw=-35.0, q_mvar=-5.0, **conv_params)

converged = pf.run_pf(net, verbose=True)
print(net.res_vsc)
```

## Appendix A: Sign Conventions

### Power Convention

| Element | Positive Power | Negative Power |
|---------|----------------|----------------|
| AC Load | Consumption | Generation |
| AC Generator | Generation | N/A |
| VSC (P_s) | Rectifier (AC→DC) | Inverter (DC→AC) |
| VSC (P_dc) | Into DC grid | Out of DC grid |

### Voltage Convention

- AC voltages are complex phasors: `V = |V| * exp(j*theta)`
- DC voltages are real positive values
- Per-unit system based on `s_base` and element rated voltages

## Appendix B: Troubleshooting

### Common Issues

**1. Power flow does not converge**
- Check that each DC grid has exactly one Vdc slack bus
- Ensure load/generation balance is reasonable
- Try increasing `max_iter_outer`
- Check VSC ratings are sufficient for required power transfer

**2. Unexpected power flow direction**
- Remember: positive P_s means rectifier (AC→DC)
- Check droop setpoints (P0 and V0)
- Verify load/generation signs

**3. DC voltage out of range**
- Adjust droop constants for better voltage regulation
- Check DC line resistances
- Consider adding reactive power support at AC buses

**4. ImportError or AttributeError from PyPower with NumPy 2.x**
- PyPower uses `numpy.in1d` which was removed in NumPy 2.0
- ACDCPF patches this automatically at import time (`np.in1d = np.isin`)
- If you see this error, make sure you `import acdcpf` before importing pypower directly
